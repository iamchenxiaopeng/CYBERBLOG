# 2026-06-19 工作日志

## 修复按钮 hover 黑色文字不可见问题

**问题**：`.btn-cyber::before` 伪元素设置了 `z-index: -1`，导致 hover 时背景色滑入层跑到父容器背景之后，背景没有实际覆盖按钮，但文字颜色 `color: #0a0a12` 是深色，在透明背景上看不见。

**修复**（`frontend/src/styles/cyber.css`）：
- `::before` 的 `z-index: -1` 改为 `z-index: 0`
- 给 `.btn-cyber` 本身加 `position: relative; z-index: 1`，确保文字始终在背景层上方
- `.btn-cyber:hover` 加 `text-shadow: none` 防止文字发光干扰可读性
- 同步修了 `.btn-cyber-pink:hover` 的相同问题

**影响范围**：全站所有 `.btn-cyber` 和 `.btn-cyber-pink` 按钮（浏览文章、进入赛博空间、登录、注册等）

**已重建并部署**：`docker compose up --build -d frontend`

## 修复头像加载失败问题

**原因**：后端返回的头像 URL 为 `https://api.dicebear.com/7.x/pixel-art/svg?seed=xxx`，外部 API 不可达时显示裂图。

**修复**：
1. 新建 `frontend/public/avatar-default.svg` — 赛博朋克风格本地 SVG 头像（霓虹边框+像素眼+扫描线）
2. 前端 3 个组件 4 处 `<img>` 加 `@error` 兜底，失败自动回退到 `/avatar-default.svg`
   - `ArticleCard.vue` 卡片作者头像
   - `ArticleDetailView.vue` 文章作者 + 评论 + 回复头像

## 按钮 hover 黑字修复（最终版）

前 3 次（z-index 方案）全部因层叠上下文/伪元素渲染问题失败。
**最终方案**：完全删除伪元素，直接在 `.btn-cyber:hover { background: var(--cyber-primary) }` 上色，100% 可靠。
