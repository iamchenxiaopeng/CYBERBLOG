### 赛博朋克博客 - 一键启动（全容器化）

**需求**：项目启动太麻烦（要分别起 docker、后端、前端），改为一行命令启动。

**方案**：将后端和前端也 Docker 化，整合进 docker-compose，加一键脚本。

**改动文件**：
1. **新增** `backend/Dockerfile`：多阶段构建（maven:3.9-eclipse-temurin-17 编译 + temurin-17-jre-alpine 运行）
2. **新增** `frontend/Dockerfile`：node:20-alpine 构建 + nginx:alpine 托管
3. **新增** `frontend/nginx.conf`：SPA fallback + `/api/` 反代到 `backend:8080`
4. **新增** `start.sh`：Linux/Mac 一键脚本（支持 prod/dev/stop/restart/logs 子命令）
5. **新增** `start.bat`：Windows 一键脚本（同上）
6. **新增** `frontend/.dockerignore` / `backend/.dockerignore`：排除不必要文件
7. **更新** `docker-compose.yml`：加入 backend + frontend 服务，配置 healthcheck 依赖链（MySQL/Redis/MinIO → backend → frontend）
8. **更新** `backend/src/main/resources/application.yml`：所有连接配置支持环境变量覆盖（`${SPRING_DATASOURCE_URL:默认值}` 格式）
9. **更新** `README.md`：启动方式改为一行命令说明

**使用方式**：
- Windows: `start.bat`
- Linux/Mac: `bash start.sh`
- 前端访问: http://localhost（nginx 80 端口）

**服务依赖顺序**：MySQL/Redis/MinIO（healthcheck）→ backend（healthcheck）→ frontend
